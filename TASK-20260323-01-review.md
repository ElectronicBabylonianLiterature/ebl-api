### 2. Filter Combinations Edge Case
**Status:** ✅ **HANDLED CORRECTLY**

```python
# Query logic in repository (CORRECT):
if centroids_only:
    annotation_filter_conditions.append(...)
if cluster_id:
    annotation_filter_conditions.append(...)
if script_filter:
    # Applied at match stage (not in filter)
    match_conditions["fragment.script.period"] = script_filter
```

This correctly handles:
- `centroids_only=True` alone → filters to centroids
- `cluster_id="X"` alone → filters to that cluster  
- `cluster_id="X" + script_filter="Neo-Assyrian"` → both filters applied ✅
- No filters → returns all annotations ✅

### 3. Exception Handling in Service
**Status:** ⚠️ **POTENTIAL ISSUE**

```python
try:
    cropped_sign_image = self._cropped_sign_image_repository.query_by_id(cropped_sign.image_id)
    # ... build response
except Exception:
    pass  # ⚠️ Silent failure
```

**Issue:** Broad exception catch silently skips annotations if image lookup fails. This is **by design** (defensive): if a cropped_sign_image is missing, skip that annotation rather than fail the entire request.

**Mitigation:** This is acceptable for now, but consider logging the failure for debugging in production.

---

### 4. Script Parameter Format
**Status:** ⚠️ **NO VALIDATION OF PERIOD VALUE**

The code passes `script_filter` directly to MongoDB query:
```python
if script_filter:
    match_conditions["fragment.script.period"] = script_filter

# Later in query: "fragment.script.period": script_filter
```

**Potential Issue:** No validation that `script_filter` is a valid period value. The query will just return no results if invalid.

**Recommendation (Optional):** Add validation or at least document expected format (e.g., "Neo-Assyrian", "Old Babylonian", etc.)
