## Key Decisions

### Decision 1: API Endpoint Design

**DECISION: Mixed Approach** (confirmed with frontend team)

```
GET /signs/{signName}/images?centroids_only=true
GET /signs/{signName}/images/cluster/{clusterId}?script={script}
```

**Rationale:**
- Query parameter for `centroids_only` (boolean filter on existing endpoint)
- Path parameter for `clusterId` (represents a specific resource/cluster)
- More RESTful: cluster is a sub-resource of sign images
- Aligns with frontend implementation expectations

**Implementation:**
- Keep existing `/signs/{sign_name}/images` route, add query param support
- Add new route: `/signs/{sign_name}/images/cluster/{cluster_id}`

---

### Decision 2: Filtering Logic Location

**Where should filtering happen?**

**Option A: Repository Layer (Recommended)**
- Filter in MongoDB aggregation pipeline using `$match` and `$filter`
- Most efficient (database does the work)
- Reduces data transfer

**Option B: Service Layer**
- Fetch all annotations, filter in Python
- Simpler to implement
- Less efficient, wastes bandwidth

**DECISION: Choose Option A** (repository layer) for performance.

```python
def find_by_sign(
    self,
    sign: str,
    centroids_only: bool = False,
    cluster_id: Optional[str] = None,
    script_filter: Optional[str] = None,
) -> Sequence[Annotations]:
    query = {"$regex": re.escape(sign), "$options": "i"}
    
    # Build match conditions
    match_conditions = {"annotations.data.signName": query}
    
    # Apply script filter if provided
    if script_filter:
        match_conditions["fragment.script.period"] = script_filter
    
    # Build annotation filter condition
    annotation_filter_cond = {"$eq": ["$$annotation.data.signName", sign]}
    
    # Apply centroids_only filter
    if centroids_only:
        annotation_filter_cond = {
            "$and": [
                annotation_filter_cond,
                {"$eq": ["$$annotation.pcaClustering.isCentroid", True]},
            ]
        }
    
    # Apply cluster_id filter
    if cluster_id:
        annotation_filter_cond = {
            "$and": [
                annotation_filter_cond,
                {"$eq": ["$$annotation.pcaClustering.clusterId", cluster_id]},
            ]
        }
    
    result = self._collection.aggregate(
        [
            {
                "$lookup": {
                    "from": "fragments",
                    "localField": "fragmentNumber",
                    "foreignField": "_id",
                    "as": "fragment",
                }
            },
            {"$unwind": "$fragment"},
            {"$match": match_conditions},
            {
                "$project": {
                    "fragmentNumber": 1,
                    "annotations": {
                        "$filter": {
                            "input": "$annotations",
                            "as": "annotation",
                            "cond": annotation_filter_cond,
                        }
                    },
                    "script": "$fragment.script",
                    "provenance": "$fragment.archaeology.site",
                }
            },
        ]
    )
    return AnnotationsWithScriptSchema().load(result, many=True, unknown=EXCLUDE)
```

**Key Changes:**
- Added optional filter parameters
- Moved `$lookup` before `$match` to enable script filtering
- Added conditional logic for `centroids_only` filter
- Added conditional logic for `cluster_id` filter
- Combined filters using `$and` when needed

from ebl.fragmentarium.application.annotations_schema import PcaClusteringSchema

class CroppedAnnotationService:
    # ... existing __init__ ...

    def find_annotations_by_sign(
        self,
        sign: str,
        centroids_only: bool = False,
        cluster_id: Optional[str] = None,
        script_filter: Optional[str] = None,
    ) -> Sequence[dict]:
        annotations = self._annotations_repository.find_by_sign(
            sign, centroids_only, cluster_id, script_filter
        )
        cropped_image_annotations = []
        
        for annotation in annotations:
            date = DateSchema().dump(
                self._fragment_repository.fetch_date(annotation.fragment_number)
            )
            annotation = attr.evolve(
                annotation,
                annotations=[
                    x
                    for x in annotation.annotations
                    if x.data.type == AnnotationValueType.HAS_SIGN
                ],
            )
            
            for annotation_elem in annotation.annotations:
                if cropped_sign := annotation_elem.cropped_sign:
                    try:
                        cropped_sign_image = (
                            self._cropped_sign_image_repository.query_by_id(
                                cropped_sign.image_id
                            )
                        )
                        
                        # Build response dict
                        response = {
                            "fragmentNumber": str(annotation.fragment_number),
                            "image": cropped_sign_image.image,
                            "script": str(annotation.script),
                            "label": cropped_sign.label,
                            "date": date,
                            "provenance": annotation.provenance,
                            "annotationId": annotation_elem.data.id,  # NEW
                        }
                        
                        # Add pcaClustering if present
                        if annotation_elem.pca_clustering:
                            response["pcaClustering"] = PcaClusteringSchema().dump(
                                annotation_elem.pca_clustering
                            )
                        
                        cropped_image_annotations.append(response)
                    except Exception:
                        pass
        
        return cropped_image_annotations
```

- Pass filters to repository
- Added `annotationId` field to response

from falcon import Request, Response
from ebl.fragmentarium.application.cropped_annotations_service import (
    CroppedAnnotationService,
)


class CroppedAnnotationsResource:
    """Handles GET /signs/{sign_name}/images with optional centroids_only filter"""
    
    def __init__(self, cropped_annotations_service: CroppedAnnotationService):
        self._cropped_annotations_service = cropped_annotations_service

    def on_get(self, req: Request, resp: Response, sign_name: str):
        # Parse query parameter
        centroids_only = req.get_param_as_bool("centroids_only", default=False)
        
        # Call service with filter
        cropped_signs = self._cropped_annotations_service.find_annotations_by_sign(
            sign_name, centroids_only=centroids_only
        )
        resp.media = cropped_signs


class ClusterCroppedAnnotationsResource:
    """Handles GET /signs/{sign_name}/images/cluster/{cluster_id}"""
    
    def __init__(self, cropped_annotations_service: CroppedAnnotationService):
        self._cropped_annotations_service = cropped_annotations_service

    def on_get(self, req: Request, resp: Response, sign_name: str, cluster_id: str):
        # Parse required query parameter
        script_filter = req.get_param("script")
        
        # Validate: script is required for cluster queries
        if not script_filter:
            resp.status = 400
            resp.media = {
                "error": "Query parameter 'script' is required for cluster queries"
            }
            return
        
        # Call service with filters
        cropped_signs = self._cropped_annotations_service.find_annotations_by_sign(
            sign_name, 
            centroids_only=False,
            cluster_id=cluster_id, 
            script_filter=script_filter
        )
        resp.media = cropped_signs
```

## API Endpoint Documentation

### GET /signs/{signName}/images

Retrieve cropped sign images for a specific sign.

**Parameters:**
- `signName` (path, required): Name of the sign to query

**Query Parameters:**
- `centroids_only` (boolean, optional): If `true`, return only centroid annotations. Default: `false`

**Examples:**
```
GET /signs/AN/images
GET /signs/AN/images?centroids_only=true
```

**Response:** Array of cropped annotation objects

**Status Codes:**
- `200 OK`: Success
- `404 Not Found`: Sign not found

---

### GET /signs/{signName}/images/cluster/{clusterId}

Retrieve cropped sign images for a specific cluster within a sign.

**Parameters:**
- `signName` (path, required): Name of the sign to query
- `clusterId` (path, required): UUID of the cluster

**Query Parameters:**
- `script` (string, required): Script/period filter (e.g., "NA", "MA", "NB")

**Examples:**
```
GET /signs/AN/images/cluster/af26e35a-1df7-56af-9b6c-30d12be2c863?script=NA
```

**Response:** Array of cropped annotation objects (centroid + all variants in cluster)

**Status Codes:**
- `200 OK`: Success
- `400 Bad Request`: Missing required `script` parameter
- `404 Not Found`: Sign or cluster not found

## Performance Considerations

### Database Query Optimization

The filtering happens in MongoDB aggregation pipeline, which is efficient:

1. **Centroids Only Query:**
   - Filters in `$filter` stage using `pcaClustering.isCentroid` field
   - Index on `annotations.pcaClustering.isCentroid` recommended for large datasets

2. **Cluster + Script Query:**
   - Filters in `$filter` stage using `pcaClustering.clusterId`
   - Script filter in `$match` stage (uses fragment script)
   - Index on `fragment.script.period` already exists

### Expected Performance Impact

Assuming typical sign has 100-200 annotations:
- **Centroids only**: Returns ~10-20 results (90% reduction)
- **Cluster query**: Returns ~5-15 results (95% reduction)

Network payload reduction: 80-95% for initial load
